# F10d.5 Fix embed CLI Stub

**Phase**: 10d - Bug Fixes & Cleanup
**Priority**: Medium
**Dependencies**: None

## Problem

The `ctx-sys embed` CLI command is a stub. It counts entities needing embeddings but prints "Use ctx-sys index to generate embeddings during indexing." without actually generating any embeddings.

### Root Cause

`src/cli/embeddings.ts generateEmbeddings()` (lines 171-174) has a placeholder comment "For now, just report what would be done" and never calls the embedding pipeline.

## Implementation

### File: `src/cli/embeddings.ts`

Wire `generateEmbeddings()` to:
1. Create EntityStore + OllamaEmbeddingProvider + EmbeddingManager
2. Load full entity objects for the identified IDs
3. Call `embeddingManager.embedIncremental()` with progress callback
4. Report results (embedded, skipped, errors)

## Success Criteria

- `ctx-sys embed --dry-run` shows entities that would be embedded
- `ctx-sys embed` actually generates embeddings via Ollama
- `ctx-sys embed --limit 10` only embeds 10 entities
- `ctx-sys embed-status` shows increased coverage after running
