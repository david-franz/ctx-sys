# F10d.1 Remove Analytics System

**Phase**: 10d - Bug Fixes & Cleanup
**Priority**: High
**Dependencies**: None

## Problem

The analytics system (token savings tracking, query logging, dashboard) produces dishonest metrics. It logs "100,000 tokens saved" when `context_query` returns 0 results. The savings percentage (90.9%) is meaningless when no context was actually retrieved. Rather than fix the metrics, the entire analytics system is removed since it adds complexity without providing genuine value.

### Observed Behavior

```
analytics_dashboard → {
  tokensSaved: 1091348,
  costSaved: 32.74,
  savingsPercent: 90.94,   // ← Meaningless: most queries returned 0 results
  averageRelevance: 0.004  // ← Confirms nothing useful was retrieved
}
```

## Implementation

### Files Deleted
- `src/analytics/` (entire directory: index.ts, types.ts, query-logger.ts, dashboard.ts, full-context-estimator.ts)
- `src/cli/analytics.ts`

### Files Modified
- `src/cli/index.ts` — remove analytics import + addCommand() calls
- `src/mcp/tool-registry.ts` — remove registerAnalyticsTools() call + method
- `src/services/core-service.ts` — remove QueryLogger, analytics methods, logQuery call in queryContext()
- `src/services/types.ts` — remove AnalyticsPeriod, AnalyticsStats, TokenSavings, DashboardData
- `src/index.ts` — remove analytics exports block

### MCP Tools Removed
- `analytics_get_stats`
- `analytics_dashboard`
- `analytics_feedback`

### CLI Commands Removed
- `ctx-sys analytics`
- `ctx-sys dashboard`

## Impact

Analytics was a cross-cutting concern that logged data but didn't affect core functionality. Removing it simplifies `queryContext()` (no more logQuery call) and removes 3 MCP tools, 2 CLI commands, and ~1000 lines of code.

## Success Criteria

- Build passes with zero TypeScript errors
- All remaining tests pass
- `analytics_get_stats`, `analytics_dashboard`, `analytics_feedback` MCP tools no longer exist
- `ctx-sys analytics` and `ctx-sys dashboard` CLI commands no longer exist
