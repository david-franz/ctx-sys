# F9.4 Product Website & User Portal

**Phase**: 9 - Analytics, Support, & Web
**Priority**: High
**Dependencies**: F9.5 NPM Package Distribution

## Goal

Create the public face of ctx-sys and the secure user portal for account management, serving as the primary acquisition channel and user interface for non-CLI interactions.

## Overview

The website serves two distinct purposes:
1. **Marketing Site**: Convert visitors into users/customers. Must explain the "Smart Librarian" concept clearly.
2. **User Portal**: Manage subscriptions, view usage stats, and access team settings.

## Tech Stack

- **Framework**: Next.js (SSG/SSR)
- **Styling**: Tailwind CSS
- **Hosting**: Azure Static Web Apps (Global Edge)
- **Content**: MDX for documentation/blog

## Key Features

### 1. Marketing Pages
- **Landing Page**: Value proposition ("Stop Repeating Yourself"), visual demo of Decision Extraction, social proof/stars.
- **Pricing Page**: Clear tier comparison (Free vs. Pro vs. Team).
- **Interactive Playground**: A WebAssembly-powered mini-demo where users can query a sample index in the browser.

### 2. Documentation Hub
- **Installation Guide**: One-line install commands.
- **Integration Guides**: Setup for VS Code, Cursor, Claude Desktop.
- **API Reference**: For enterprise users integrating programmatically.

### 3. User Portal (Dashboard)
- **Overview**: Active plan, license keys, usage metrics.
- **Team Management**: Invite members, assign roles (Admin, Member).
- **Billing**: Invoice history, payment method update (via Stripe Customer Portal).
- **Download Center**: Links to latest Desktop App releases (.dmg, .exe).

## Implementation Plan

1. **Scaffold Next.js Project**: configure `app` directory structure.
2. **Design System**: Implement basic brand identity (colors, typography).
3. **Integration**: Connect to Auth provider (F10.2) and Billing provider (F10.5).
4. **Content**: Migrate relevant parts of `docs/*.md` to public documentation.
