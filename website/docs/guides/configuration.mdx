# Configuration

ctx-sys uses a YAML configuration file to customize behavior for your project.

## Configuration File

The main configuration file is `.ctx-sys/config.yaml` in your project root.

```yaml
project:
  name: my-project
  description: My awesome project

indexing:
  mode: incremental
  watch: true
  ignore:
    - node_modules
    - dist
    - "**/*.test.ts"
    - "**/*.spec.ts"
  languages:
    - typescript
    - python

embeddings:
  provider: ollama
  model: nomic-embed-text

summarization:
  enabled: true
  provider: ollama
  model: qwen3:0.6b

retrieval:
  default_max_tokens: 4000
  strategies:
    - vector
    - graph
    - fts
```

## Configuration Options

### Project Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `name` | string | directory name | Project identifier |
| `description` | string | `""` | Project description |

### Indexing Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `mode` | string | `"incremental"` | Indexing mode |
| `watch` | boolean | `false` | Enable file watching |
| `ignore` | string[] | common patterns | Glob patterns to exclude |
| `languages` | string[] | `["all"]` | Languages to index |

### Embedding Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `provider` | string | `"ollama"` | Provider: ollama, openai |
| `model` | string | `"nomic-embed-text"` | Embedding model name |

### Summarization Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `enabled` | boolean | `true` | Enable LLM summaries |
| `provider` | string | `"ollama"` | Provider: ollama, openai, anthropic |
| `model` | string | `"qwen3:0.6b"` | Summarization model |

### Retrieval Settings

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `default_max_tokens` | number | `4000` | Token budget for context |
| `strategies` | string[] | `["vector", "graph", "fts"]` | Search strategies |

## Environment Variables

Some settings can be overridden via environment variables:

```bash
# Embedding provider API key
export OPENAI_API_KEY=sk-...

# Anthropic API key (for cloud summarization)
export ANTHROPIC_API_KEY=sk-ant-...

# Ollama server URL (if not default)
export OLLAMA_HOST=http://localhost:11434

# Database path (override default)
export CTX_SYS_DB_PATH=/path/to/database.db

# Log level
export CTX_SYS_LOG_LEVEL=debug
```

## Global Configuration

Global settings (applying to all projects) are stored in:

- **macOS/Linux**: `~/.ctx-sys/config.yaml`
- **Windows**: `%USERPROFILE%\.ctx-sys\config.yaml`

```yaml
database:
  path: ~/.ctx-sys/ctx-sys.db

providers:
  ollama:
    base_url: http://localhost:11434
  openai:
    api_key: ${OPENAI_API_KEY}
```

## Recommended Configurations

### TypeScript Project

```yaml
project:
  name: my-ts-project

indexing:
  languages:
    - typescript
  ignore:
    - node_modules
    - dist
    - "**/*.test.ts"
    - "**/*.spec.ts"
    - "**/*.d.ts"
```

### Python Project

```yaml
project:
  name: my-python-project

indexing:
  languages:
    - python
  ignore:
    - venv
    - __pycache__
    - "test_*.py"
    - "*_test.py"
```

### Monorepo

```yaml
project:
  name: my-monorepo

indexing:
  ignore:
    - node_modules
    - dist
    - build
```

## Validation

Validate your configuration:

```bash
ctx-sys health
```

View project status:

```bash
ctx-sys status
```
