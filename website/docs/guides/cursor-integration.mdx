# Cursor Integration

This guide explains how to use ctx-sys with Cursor IDE.

## Overview

Cursor has built-in AI capabilities, but ctx-sys can enhance them by providing:
- Better context retrieval from your codebase via hybrid search
- Persistent conversation memory
- Decision tracking across sessions
- Smart context expansion with parent classes and imports

## Setup Options

### Option 1: MCP Server (Recommended)

If Cursor supports MCP servers, configure ctx-sys as an MCP server:

```json
{
  "mcp": {
    "servers": {
      "ctx-sys": {
        "command": "ctx-sys",
        "args": ["serve"],
        "cwd": "${workspaceFolder}"
      }
    }
  }
}
```

### Option 2: HTTP API

Run ctx-sys as an HTTP server:

```bash
ctx-sys serve --http --port 3000
```

Then configure Cursor to use the HTTP endpoint for context retrieval.

### Option 3: File-Based Context

Generate context files that Cursor can include:

```bash
# Generate a context snapshot
ctx-sys search "How does authentication work?" > .cursor/context.md

# Update context in .cursorrules
echo "Include context from .cursor/context.md" >> .cursorrules
```

## Best Practices

### 1. Use .cursorrules

Add ctx-sys context to your `.cursorrules` file:

```markdown
# Project Context

This project uses:
- TypeScript with strict mode
- Express.js for the API
- PostgreSQL for the database

## Key Decisions

Run `ctx-sys search-decisions` to see recent architectural decisions.

## Getting Context

For any code changes, first ask ctx-sys:
`ctx-sys search "your question"`
```

### 2. Context Commands

Add useful context commands to your workflow:

```bash
# Add to package.json scripts
{
  "scripts": {
    "ctx:auth": "ctx-sys search 'authentication system'",
    "ctx:api": "ctx-sys search 'API patterns'",
    "ctx:db": "ctx-sys search 'database schema'"
  }
}
```

### 3. Git Hooks Integration

Use ctx-sys git hooks to keep context fresh:

```bash
ctx-sys hooks-install
```

This automatically re-indexes changed files on commit.

## Tips

1. **Keep context focused** - Request specific context rather than "everything about X"

2. **Use decisions** - Run `ctx-sys search-decisions` to understand past choices

3. **Index before sessions** - Run `ctx-sys index` before starting a coding session

## Troubleshooting

### Cursor not finding context

Ensure your project is indexed:

```bash
ctx-sys status
ctx-sys index
```

### Context seems irrelevant

Try more specific queries:

```bash
# Instead of:
ctx-sys search "database"

# Use:
ctx-sys search "User table schema and relationships"
```
