# Introduction to ctx-sys

**ctx-sys** is an intelligent context management system for AI-assisted coding. It acts as a "smart librarian" for your codebase, providing AI tools like Claude, GPT, and Copilot with precisely the context they need - nothing more, nothing less.

## The Problem

When using AI coding assistants, you face a dilemma:

- **Too little context**: The AI doesn't understand your codebase, leading to incorrect or irrelevant suggestions
- **Too much context**: Sending your entire codebase burns tokens, costs money, and often confuses the AI

Traditional approaches force you to manually copy-paste relevant files or hope the AI figures out what's relevant. This is time-consuming and error-prone.

## The Solution

ctx-sys solves this by:

1. **Indexing your codebase** - Understanding your code structure, relationships, and patterns via AST parsing
2. **Remembering conversations** - Tracking decisions, context, and history across sessions
3. **Retrieving intelligently** - Using hybrid search (semantic + graph + FTS5 keyword) to find exactly what's relevant
4. **Saving tokens** - Typically reducing context size by 60-80% while maintaining relevance

## Key Features

### Intelligent Code Understanding

- **AST parsing** for TypeScript, JavaScript, Python, and more via tree-sitter
- **Automatic relationship extraction** - imports, calls, extends, implements discovered from AST
- **AI-generated summaries** via Ollama (qwen3:0.6b) or cloud providers
- **Smart context expansion** - methods automatically include their parent class, functions include their imports

### Hybrid RAG Retrieval

- **Multi-strategy search** combining FTS5 keyword, semantic vector, and graph traversal
- **Reciprocal Rank Fusion** to merge results from all strategies
- **Query decomposition** for complex multi-part questions
- **LLM re-ranking** to score relevance of top candidates
- **HyDE query expansion** for better semantic matching

### Universal Document Indexing

- **Code files** - TypeScript, JavaScript, Python via tree-sitter AST parsing
- **Documents** - Markdown, YAML, JSON, TOML, plain text
- **Incremental updates** - Hash-based change detection skips unchanged files
- **Overlapping chunks** - Long entities are split at smart boundaries (paragraph, line, sentence) with overlap to prevent information loss

### Conversation Memory

- **Session persistence** across IDE restarts
- **Decision tracking** - never lose track of "why" decisions
- **Context carry-over** between related conversations

### Privacy & Local-First

- **No cloud required** - Run entirely locally with Ollama
- **Your data stays yours** - Single SQLite file (better-sqlite3), no external services
- **Optional cloud** - Use OpenAI/Anthropic only if you choose to

### Developer Experience

- **30+ CLI commands** covering all functionality
- **MCP integration** for seamless Claude Desktop/Code connection
- **Real-time indexing** with file watching
- **Git hooks** for automatic indexing on commit

## Quick Start

Get started in under 2 minutes:

```bash
# Install globally
npm install -g ctx-sys

# Initialize in your project
cd your-project
ctx-sys init

# Index your codebase
ctx-sys index

# Start the MCP server
ctx-sys serve
```

Then connect to Claude Desktop and start asking questions about your codebase!
